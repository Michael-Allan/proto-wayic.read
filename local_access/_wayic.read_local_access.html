Web access to the local wayscript

  1. Copy this file to the base directory of your waycast
  2. Load it into your browser, and from there:
  3. Follow this link: visionary/way.xht

Supported browsers

Firefox. See the list of browsers tested, and see below.

Why this is needed

When rendering local wayscript (loaded from a ‘file’ scheme URL, that is), wayic.read has a limited ability to resolve external waylinks (marked ‘⌚’ till resolved), to trace the way across multiple files, or generally to read other way files. It’s possible to work around this limitation (depending on the browser) with the help of this support file.

It doesn’t work on Chrome. Chrome (59) rejects all local read requests, logging the reason in the console as “Cross origin requests are only supported for protocol schemes: http, data, chrome, chrome-extension, https.” Firefox (52) is more lenient. It forwards at least the in-branch requests, viz. where “the parent directory of the originating file is an ancestor directory of the target file.” *
